# プロジェクト詳細ページ マルチビュー実装

## Overview

プロジェクト詳細ページに3つのビューを追加する（現在「準備中」で disabled）:

1. **カードビュー** — タスクをギャラリー形式で表示（Notion Gallery風）
2. **看板ビュー** — ステータス別カラムにタスクを配置（Notion Board風）
3. **タイムラインビュー** — セッション＋タスクを時系列で表示（Notion Timeline風）

現在のダッシュボードビューはそのまま維持し、ヘッダーのアイコンボタンで切り替える。

## 技術方針

### ビュー状態管理

- `ProjectDetailPage.tsx` に `viewMode` signal を追加（`'dashboard' | 'card' | 'kanban' | 'timeline'`）
- `ProjectHeader` に `onViewChange` コールバックを渡してビュー切り替え
- ビュー状態は `localStorage` で永続化（プロジェクト横断で最後に使ったビューを記憶）

### コンポーネント構成

```
ProjectDetailPage.tsx（オーケストレータ）
├── ProjectHeader.tsx（ビュー切り替えボタン修正）
├── SummaryCards.tsx（全ビュー共通）
├── DashboardView（既存: Active/Tasks/Paused/Done セクション群）
├── CardView.tsx（新規）
├── KanbanView.tsx（新規）
└── TimelineView.tsx（新規）
```

### データ

新規APIは不要。既存ストアのデータをフィルタして使用:
- `projectTodos` — タスク一覧（status, priority, due_date, tags, parent_id）
- `projectSessions` — セッション一覧（status, created_at, updated_at, linked_tasks）

## 各ビューの仕様

### 1. カードビュー（Gallery）

**レイアウト**: CSS Grid（`grid-cols-1 sm:grid-cols-2 lg:grid-cols-3`）

**カード内容**:
```
┌──────────────────────┐
│ [チェック] タスク名    │
│ !!!  #tag1  #tag2     │
│                       │
│ 期限: 2026-03-15      │
│ サブタスク: ██░░ 2/4  │
└──────────────────────┘
```

- 優先度バッジ（!!!、!!）
- タグ一覧
- 期限表示（due_date がある場合、超過なら赤）
- サブタスク進捗バー（子タスクがある場合）
- チェックボックスでステータス切り替え
- カードクリックでインライン編集

**フィルタ/ソート**: デフォルトはステータス→優先度順。完了タスクは末尾に薄く表示。

### 2. 看板ビュー（Kanban Board）

**レイアウト**: 横スクロール可能な3カラム

```
┌─────────────┬─────────────┬─────────────┐
│  未着手 (5)  │  進行中 (2)  │  完了 (10)  │
├─────────────┼─────────────┼─────────────┤
│ ┌─────────┐ │ ┌─────────┐ │ ┌─────────┐ │
│ │ タスクA  │ │ │ タスクC  │ │ │ タスクE  │ │
│ │ !!! #tag │ │ │ !! #tag  │ │ │ ✓ done  │ │
│ └─────────┘ │ └─────────┘ │ └─────────┘ │
│ ┌─────────┐ │             │ ┌─────────┐ │
│ │ タスクB  │ │             │ │ タスクF  │ │
│ └─────────┘ │             │ └─────────┘ │
│             │             │ ... +8 more │
│ [+ 追加]    │             │             │
└─────────────┴─────────────┴─────────────┘
```

- 3カラム: pending / in_progress / completed
- カラムヘッダーにタスク数表示
- 完了カラムは最初の5件表示 + "もっと見る"
- **D&D**: カード移動でステータス変更（HTML5 Drag & Drop API — MatrixView の既存パターンを踏襲）
- 未着手カラムに「+ 追加」ボタン
- カード内容はカードビューと同じ構造（ただしコンパクト版）

### 3. タイムラインビュー（Timeline）

**レイアウト**: 垂直タイムライン

```
2026-03-03
──────────────────────────────
  ● セッション「UI実装」 (active)  ← 緑ドット
  │  タスクA ✓
  │  タスクB (進行中)
  │
  ○ タスクC 追加                   ← グレードット

2026-03-02
──────────────────────────────
  ● セッション「設計」 (done)      ← グレードット
  │  タスクD ✓
  │  タスクE ✓
  │
  ○ タスクF 追加
```

- セッションとタスクを `created_at` / `updated_at` の日付でグループ化
- セッション: アクティブ=緑、一時停止=黄、完了=グレー
- タスク: 作成日をイベントとして表示
- 日付の降順（新しい順）
- 直近7日間をデフォルト表示、「もっと読み込む」で拡張

## Acceptance Criteria

### ビュー切り替え
- [ ] ヘッダーの4つのアイコンボタンがすべてクリック可能
- [ ] アクティブなビューのボタンが `bg-app-accent text-white` でハイライト
- [ ] ビュー切り替え時にアニメーションなし（即時切り替え）
- [ ] 選択ビューが `localStorage` に保存され、再訪時に復元

### カードビュー
- [ ] タスクがグリッド形式でカード表示される
- [ ] チェックボックスでステータス切り替え可能
- [ ] 優先度・タグ・期限・サブタスク進捗が表示される
- [ ] レスポンシブ（モバイル1列、タブレット2列、デスクトップ3列）

### 看板ビュー
- [ ] 3カラム（未着手・進行中・完了）でタスク表示
- [ ] D&Dでカラム間移動 → ステータス自動更新
- [ ] 完了カラムは5件まで表示 + 展開可能
- [ ] 未着手カラムにタスク追加機能

### タイムラインビュー
- [ ] セッション＋タスクが日付グループで時系列表示
- [ ] セッションステータスが色分け表示
- [ ] デフォルト7日間、もっと読み込み可能

### 共通
- [ ] `SummaryCards` は全ビューで表示
- [ ] 既存ダッシュボードの動作に影響なし
- [ ] ダーク背景でテキストが正しく表示される

## 実装順序

1. **ビュー切り替え基盤** — `ProjectHeader` + `ProjectDetailPage` の signal/localStorage
2. **カードビュー** — 最もシンプル、既存 `TodoItem` を参考にカード化
3. **看板ビュー** — D&D付き、MatrixViewの `drag-over-cell` パターン流用
4. **タイムラインビュー** — データ加工が必要、最後に実装
