# セッションとタスクの関連付け

## 課題

現状、work_sessionsとtodosは `project` 文字列の一致以外に関係がない。セッションで何のタスクを進めたか、タスクがどのセッションで作業されたかが見えない。

## 何を作るか

`session_tasks` 中間テーブルで多対多の紐付けを実現。セッション詳細から関連タスクを表示し、タスク側からも関連セッションを表示する。ダッシュボードではセッションごとのタスク進捗を俯瞰できる。

## なぜこのアプローチか

- 多対多の関係（同じタスクを複数日のセッションで進める）を自然に表現できる
- 双方向参照が容易（JOINで両方向から取得可能）
- ダッシュボードでの進捗集計（完了率など）がSQLで簡単にできる
- タグ方式やログ内リンクより構造化されており壊れにくい

## キー判断

| 項目 | 決定 | 理由 |
|------|------|------|
| 関係モデル | 多対多（中間テーブル） | 1タスクを複数セッションで進めるケースがある |
| 紐付けタイミング | 作業中に都度 | セッション詳細画面から「タスクを紐付け」で追加 |
| 表示方向 | 双方向 | セッション→タスク一覧、タスク→セッション履歴 |
| ダッシュボード | セッションカードにタスク進捗表示 | 完了数/全体数 でざっくり把握 |
| 紐付け解除 | 可能 | 誤って紐付けた場合に外せる |
| 中間テーブル | session_tasks (session_id, todo_id) | シンプルな紐付けのみ。メモ等は不要 |
| 既存project関係 | 維持（独立） | projectは大きな分類、session_tasksは明示的な紐付け |

## ユーザー体験

### セッション詳細画面
- 関連タスク一覧（チェックボックス付き、完了操作可能）
- 「タスクを紐付け」ボタン → タスク選択UI
- タスクの紐付け解除

### タスク側
- タスク詳細 or タスクメタ情報に関連セッションを表示

### ダッシュボード
- セッションカードに `3/5タスク完了` のような進捗表示

## 解決済みの問題

| 問題 | 決定 |
|------|------|
| タスク選択UIの形式 | インライン検索（セッション詳細内にテキスト入力でタスクを検索・選択） |
| タスク側の関連セッション表示 | タスクのメタ情報に追加（日付やプロジェクトの横にセッション名を表示） |
| CLIからの紐付け操作 | 必要（`/taskflow session link` コマンドを追加） |
