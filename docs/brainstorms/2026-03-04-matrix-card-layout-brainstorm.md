# ファーストビュー: マトリックスビュー → カード型レイアウト改修

**Date:** 2026-03-04
**Status:** Final

## What We're Building

マトリックスビュー（ファーストビュー）を現在の5カラムグリッドテーブルから**カード型縦並びリスト**に刷新する。CMUX連携はフェーズ2として後追い実装する。

### 目的

1. **Task・Active・Project** の3要素を大きく、明確に見せる
2. Paused/Doneが場所を取りすぎる問題を解消
3. 「今何をやっているか」が一目でわかるファーストビューにする

## Why This Approach

### カード型を選んだ理由

- 5カラムのグリッドテーブルは全カラムが等幅で、重要度の差が視覚的に表現できない
- カード型なら各プロジェクトの情報密度を動的に変えられる（Active有 → 大、無 → 小）
- Paused/Doneをバッジ化して省スペース化できる

### 検討した代替案

| 案 | 概要 | 不採用理由 |
|----|------|-----------|
| A: 3カラム主体 | Project/Active/Tasksの3列に縮小 | テーブル構造が残り、根本的な見た目の変化が小さい |
| B: 可変幅5カラム | Active/Tasksを広く、Paused/Doneを狭く | 5カラムのままでインパクト不足 |

## Key Decisions

### 1. レイアウト構造

```
┌──────────────────────────────────────────────────┐
│ ● Project Alpha              [Paused:1] [Done:3] │
│ ┌──────── Active ────────┐  ┌──── Tasks ────┐   │
│ │ ▶ UI改修セッション  🔗  │  │ ☐ タスク1     │   │
│ │   最新ログ: CSS修正中.. │  │ ☐ タスク2     │   │
│ │   ████████░░ 6/10      │  │ ☑ タスク3     │   │
│ └────────────────────────┘  └───────────────┘   │
└──────────────────────────────────────────────────┘
┌──────────────────────────────────────────────────┐
│ ● Project Beta（Active無し・コンパクト表示）       │
│   Tasks: 3件  Paused:0  Done:2                   │
└──────────────────────────────────────────────────┘
```

- **全幅カード、縦並び**
- カード内は横2カラム: Active（左・大）+ Tasks（右）
- Paused/Doneはカードヘッダーにバッジとして件数表示

### 2. Active セッションの表示

- セッションタイトルを大きく表示
- 最新ログのプレビュー（1行）
- 進捗バー（task_completed / task_total）
- クリックで既存のインライン詳細パネルを展開（現行動作維持）

### 3. 非アクティブプロジェクトのコンパクト表示

- Activeセッションがないプロジェクトは1行程度のコンパクトカードに
- Tasks件数とPaused/Doneバッジだけ表示
- クリックで展開可能（タスク一覧等）

### 4. Paused/Done の扱い

- 独立カラム廃止 → カードヘッダーにバッジ（件数のみ）
- バッジクリックでカード内インライン展開

### 5. カードのソート順

- **Activeセッション有りのプロジェクトを上に配置**
- Active有り内では更新順
- Active無しは下にまとめてコンパクト表示

### 6. CMUX ワークスペース連携（フェーズ2・別タスク）

- **アーキテクチャ**: ブラウザ → ローカルHTTPプロキシ → Unix Socket(`/tmp/cmux.sock`)
- **データモデル**: `work_sessions`テーブルに`workspace_id`カラム追加
- **UI**: Activeセッションのタイトル横にアイコン配置
- **動作**: アイコンクリックで`localhost:PORT`経由で`workspace.select` API呼び出し
- **クリック動作の分離**: カードクリック=詳細展開、CMUXアイコン=ワークスペース切替
- **紐付け**: セッション作成/編集時にワークスペースIDを入力可能に
- **CMUXにはURLスキーム無し**: `cmux://`プロトコルは存在しない。Unix Socket or CLI経由のみ

### 7. 既存機能の維持

- タグフィルター、アーカイブトグルは維持
- プロジェクト追加フォームは維持
- タスクのD&D（ドラッグ＆ドロップ）は維持
- セッションのインライン詳細パネルは維持

## Resolved Questions

1. **CMUXのURLスキーム** → URLスキーム無し。Unix Socket(`/tmp/cmux.sock`)またはCLI経由。ローカルHTTPプロキシで中継する方針
2. **ソート順** → Activeセッション有りのプロジェクトを上に優先表示
3. **Paused/Doneバッジの展開** → カード内インライン展開
