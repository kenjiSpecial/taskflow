# メニューバーアプリ: セッション優先表示 + cmuxワークスペース連携

## What We're Building

メニューバーアプリのメインパネルを改善し、アクティブセッションをプロジェクト単位で最上部に表示。cmuxワークスペースの状態を可視化し、直接開く/完了する操作を提供する。

### 表示レイアウト（優先順位順）

```
┌─────────────────────────────────┐
│ TaskFlow                    ⟳ ⚙ │
├─────────────────────────────────┤
│ 🟢 taskflow (プロジェクト名)     │
│   ├ OpenClaw認証リファクタ  [開く]│
│   └ APIテスト            [WS作成]│
│                                 │
│ 🟡 openclaw (プロジェクト名)     │
│   └ JWT移行セッション    [WS作成]│
│                                 │
│ 📅 今日のタスク                  │
│   ├ ● セッション検索機能の追加    │
│   └ ● ダッシュボードのレスポンシブ │
│                                 │
│ ▶ 進行中                        │
│   └ (なし)                      │
├─────────────────────────────────┤
│ Taskflowを開く              ⚙   │
└─────────────────────────────────┘
```

## Why This Approach

- **セッション中心**: 開発者の「今やっていること」をすぐ確認できる
- **プロジェクトグルーピング**: どのプロジェクトの作業かが一目瞭然
- **ネイティブ直接実行**: ブリッジサーバー不要、`Process`で`taskflow-cmux`コマンドを直接叩く

## Key Decisions

1. **表示順序**: アクティブセッション → 今日のタスク → 進行中タスク
2. **セッションのソート**: WS開いている（mappings.jsonに存在）ものが上
3. **ワークスペース操作**: ネイティブで直接実行（ブリッジサーバー経由しない）
   - 開く/作成: `taskflow-cmux start <session-id>` をProcess実行
   - 完了: `taskflow-cmux stop <session-id>` をProcess実行
4. **WS状態検知**: `~/.taskflow-cmux/mappings.json` を直接読み取り（ブリッジが動いていなくても確認可能）
5. **セッション操作スコープ**: 開く + 完了のみ（一時停止はcmuxから直接）
6. **プロジェクト名表示**: セッションの`project`フィールドまたは`project_id`経由でプロジェクト名を取得
