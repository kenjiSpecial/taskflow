# 統合マトリクスビュー ブレインストーム

**日付**: 2026-03-02
**ステータス**: ブレインストーム完了

## What We're Building

タスク・セッション・プロジェクトを1つの「マトリクスビュー」に統合する。

- **縦軸**: プロジェクト（+ 未分類行）
- **横軸**: セッションステータス（Active / Paused / Done）+ 未リンクタスク列
- **セル内**: セッションカード（タイトル + 進捗バー）とタスク一覧
- 既存のViewToggle（タスク/セッション切り替え）を廃止し、このマトリクスビュー1本に置き換える

```
┌─────────────────────────────────────────────────────┐
│  [+ プロジェクト追加]           [アーカイブ表示 ☐]    │
├──────────┬──────────┬──────────┬──────────┬─────────┤
│          │ Active   │ Paused   │ Done     │ タスク   │
├──────────┼──────────┼──────────┼──────────┼─────────┤
│ PJ-A [⚙]│ 🟢S1    │          │ ✅S3     │ 3未完了  │
```

## Why This Approach

1. **一目で全体像把握**: プロジェクト × セッションステータスのグリッドで、何が動いているか瞬時にわかる
2. **動的な状態が目立つ**: Active列に注目するだけで「今何が進行中か」が明確
3. **既存の分断を解消**: タスクビューとセッションビューが別々だった問題を統合
4. **D&Dによるプロジェクト変更**: タスクを別行にドラッグするだけでプロジェクト紐付け変更

## Key Decisions

### 1. プロジェクトを専用テーブル化

**現状**: `project`フィールドは単なるstring（FK無し）
**変更**: `projects`テーブルを新設

```sql
CREATE TABLE projects (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL UNIQUE,
  description TEXT,
  color TEXT,
  archived_at TEXT,
  created_at TEXT NOT NULL DEFAULT (datetime('now')),
  updated_at TEXT NOT NULL DEFAULT (datetime('now')),
  deleted_at TEXT
);
```

- `todos.project` → `todos.project_id`（FK参照）に変更
- `work_sessions.project` → `work_sessions.project_id`（FK参照）に変更
- マイグレーションで既存stringデータをprojectsレコードに変換

### 2. プロジェクトCRUD + アーカイブ

- **追加**: プロジェクト名（必須）+ description, color（任意）
- **編集**: 名前・説明・カラー変更
- **アーカイブ**: `archived_at`をセット。マトリクスから行ごと非表示。トグルで再表示可能
- **削除**: 論理削除（`deleted_at`）。紐付くタスク・セッションのproject_idはnullに

### 3. ViewToggle廃止 → マトリクスビュー1本

- 既存の「タスク / セッション」切り替えを廃止
- マトリクスビューがデフォルトかつ唯一のビュー
- ProjectSidebarも廃止（プロジェクトはマトリクスの縦軸で表現）

### 4. タスクのプロジェクト変更はD&D

- タスクを別プロジェクト行にドラッグ&ドロップしてproject_id変更
- 既存のタスク並び替えD&Dと共存（同一行内=並び替え、行跨ぎ=PJ変更）

### 5. セッション詳細はインライン展開

- セッションカードをクリック → その場で詳細展開（現在のSessionDetailの内容）
- ログ追加、タスクリンク、ステータス変更がインラインで可能

### 6. 「タスク」列（右端）

- 各プロジェクトの未完了タスク数を表示
- 展開するとタスク一覧（既存TodoListの機能）
- タスク作成フォームもここに配置
- セッションに紐付いていないタスクも含む

## Scope

### In Scope
- `projects`テーブル新設 + マイグレーション（既存データ移行含む）
- Projects API: CRUD + アーカイブ
- マトリクスビュー（メイン画面置き換え）
- プロジェクト追加・編集・アーカイブ・削除UI
- タスクのプロジェクトD&D変更
- セッションカード表示（タイトル + 進捗バー）
- タスク列（展開でタスク一覧）
- セッション詳細のインライン展開

### Out of Scope
- プロジェクト間のセッションD&D移動（セッションのPJ変更は編集で対応）
- プロジェクトの並び替え（アルファベット順固定）
- モバイルレスポンシブ対応（デスクトップ優先）
- タスクのフィルタバー（マトリクスでプロジェクトフィルタ不要に）

## Resolved Questions

1. **Doneセッションの表示制限**: Done列は最新3件のみ表示。「もっと見る」で展開。
2. **未分類行の扱い**: project_id=nullのタスク/セッションが存在する場合のみ「(未分類)」行を表示。
3. **既存データのマイグレーション**: 大文字小文字の揺れはそのまま移行。別プロジェクトとして扱う。問題があれば手動修正。
