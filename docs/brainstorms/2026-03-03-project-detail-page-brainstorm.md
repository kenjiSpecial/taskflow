# Brainstorm: プロジェクト個別ページの作成

**Date:** 2026-03-03
**Status:** Draft

## What We're Building

プロジェクト個別ページ — プロジェクトに紐づくタスクとセッションの状態を、ダッシュボード形式で一目で把握・操作できるページ。

### 要件

- URLベースの直接アクセス（`/#/projects/:id`）
- デフォルトビューは**ダッシュボード形式**（サマリー + セクション分割）
- 将来的にカード・看板・タイムラインビューへの切り替えを想定
- タスク・セッションのフルCRUD操作が可能
- 既存のMatrixViewと並列共存（MatrixView = 全体俯瞰、個別ページ = 深掘り）

### ユーザーが知りたいこと

1. **今何がアクティブで、何が未着手か** — ステータス別にグループ化して視認性を確保
2. **一覧形式ではなく、簡潔かつ詳細に** — サマリーカードで概要把握 + セクション内で詳細表示

## Why This Approach

### preact-router導入 + ダッシュボードファースト

- `preact-router` でハッシュルーティングを導入
- `/#/` = MatrixView（既存）、`/#/projects/:id` = ProjectDetailPage（新規）
- ダッシュボードを最初のビューとして実装し、後からビュー追加可能な構造にする

**選定理由:**
- URL直接アクセスとMatrixViewからの遷移の両方をサポート
- ルーターがあれば将来のページ追加（タスク詳細ページ等）も容易
- ダッシュボードはユーザーの主要ニーズ（アクティブ/未着手の把握）に最も合致

## Key Decisions

### 1. ルーティング

- **preact-router** を採用（Preactエコシステム内で軽量）
- ハッシュベース（`/#/`）でCloudflare Pagesのルーティング設定不要
- URL構造: `/#/` (Matrix), `/#/projects/:id` (プロジェクト詳細)

### 2. ページ構成（ダッシュボードビュー）

```
┌─────────────────────────────────────────────────┐
│ ← MatrixView   プロジェクト名        [ビュー切替] │
├─────────────────────────────────────────────────┤
│  [アクティブ: N]  [一時停止: N]  [タスク: N/M]   │  ← サマリーカード
├─────────────────────────────────────────────────┤
│                                                 │
│  ■ アクティブセッション                           │  ← セクション1
│    セッション名 | 経過時間 | 紐づきタスク          │
│    [一時停止] [完了]                              │
│                                                 │
│  ■ タスク                                       │  ← セクション2
│    ステータス別グループ                            │
│    (進行中 → 未着手 → 完了)                       │
│    各タスク: チェックボックス、名前、優先度、タグ    │
│    [+ タスク追加]                                 │
│                                                 │
│  ■ 一時停止中セッション                           │  ← セクション3
│    セッション名 | 停止時間 | [再開] [完了]         │
│                                                 │
│  ■ 完了セッション（折りたたみ）                    │  ← セクション4
│    最近N件表示                                    │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 3. 操作範囲

- タスク: 作成、編集、ステータス変更、削除、並び替え
- セッション: 作成、開始/一時停止/完了、ログ追加、タスク紐づけ
- 既存のストア関数をそのまま活用（project-store, todo-store, session-store）

### 4. ビュー切り替え

- ヘッダーにビュー切り替えボタンを配置
- 初期実装はダッシュボードのみ、ボタンは将来の拡張ポイントとして構造だけ用意
- ビュー状態はURL（クエリパラメータ等）またはローカルストレージで保持

### 5. スタイリング

- **Tailwind CSS** を新規導入
- 新ページはTailwindで実装、既存MatrixViewの素CSSはそのまま共存
- 既存のCSS変数（`--bg`, `--surface`, `--accent`等）をTailwindのカスタムカラーに統合
- 段階的に既存コンポーネントもTailwind化する可能性あり

### 6. データ取得

- 既存API `/api/projects/:id` でサマリーデータ取得
- `/api/todos?project_id=:id` でタスク一覧
- `/api/sessions?project_id=:id` でセッション一覧（要API確認：project_idフィルタの有無）
- 既存ストアのデータをフィルタして使うか、専用のフェッチ関数を追加するか要検討

## Resolved Questions

1. **セッションAPIにproject_idフィルタはあるか？** → あり。`GET /api/sessions?project_id=:id` で対応済み
2. **MatrixViewからプロジェクト個別ページへの遷移UI** → **プロジェクト名クリック**で遷移
3. **ビュー切り替えのUI具体形** → **アイコンボタン**をヘッダーに並べる（Notion風）
4. **スタイリング** → **Tailwind CSS**を新規導入。既存素CSSと共存
